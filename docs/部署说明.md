# KIBot 部署说明

## 本地开发部署

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 7860 --reload
```

前端：

```bash
cd frontend
npm install
npm run dev
```

访问：

- 前端：`http://localhost:5173`
- API：`http://localhost:7860`
- OpenAPI：`http://localhost:7860/docs`

## 环境变量

建议从 `.env.example` 复制：

```bash
cp .env.example .env
```

必须注意：

- `OPENAI_API_KEY` 使用占位值或部署环境变量，不写入仓库。
- `SESSION_STORAGE_DIR` 在生产环境应挂载到持久化目录。
- `OPENAI_BASE_URL` 可指向任意兼容 Chat Completions 的服务。

## Docker 部署要点

仓库已提供 Dockerfile 与 docker-compose 配置。Docker 镜像会先构建 React 前端，再启动 FastAPI，并由 `app.py` 在存在 `frontend/dist` 时托管前端静态文件。

部署原则：

1. 后端进程监听 `0.0.0.0:7860`。
2. 使用 `--env-file .env`、`docker compose` 自动读取的 `.env` 变量，或平台密钥管理注入 LLM 配置。
3. 将宿主机 `data` 目录或平台持久化目录挂载到容器中的 session storage 路径。
4. 不把 `.env`、上传教材、session JSON 打包进镜像。

示例：

```bash
docker build -t kibot .
docker run --env-file .env -p 7860:7860 -v "$PWD/data:/app/data" kibot
```

使用 compose：

```bash
docker compose up --build
```

评审的一键部署口径：

- 一条命令：`docker compose up --build`
- 对外端口：`7860`
- 健康检查：`GET http://localhost:7860/api/health`
- 浏览器入口：`http://localhost:7860/`
- 数据持久化：`SESSION_STORAGE_DIR=/app/data/sessions`，由 compose 挂载本地 `data` 目录
- 前端静态资源：Docker 构建阶段执行 `npm ci && npm run build`，运行阶段由 FastAPI 托管 `frontend/dist`

魔搭创空间建议：

- 运行端口：`7860`
- 启动方式：Dockerfile
- 环境变量：`OPENAI_BASE_URL`、`OPENAI_API_KEY`、`OPENAI_MODEL`
- 持久化目录：`/app/data`

## 评审演示建议

1. 打开前端仪表盘。
2. 创建 session。
3. 上传或选择 7 本教材。
4. 选择教材并构建图谱。
5. 点击“整合到30%”，展示压缩率、Sankey 整合流和决策卡片。
6. 在教师对话中输入复核意见，展示多轮迭代能力。
7. 在 RAG 中提问，展示引用的教材、章节和页码。
8. 预览 Markdown 实例报告。
