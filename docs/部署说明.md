# KIBot 部署说明

## 本地开发部署

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 7860 --reload
```

前端：

```bash
cd frontend
npm install
npm run dev
```

访问：

- 前端：`http://localhost:5173`
- API：`http://localhost:7860`
- OpenAPI：`http://localhost:7860/docs`

## 环境变量

建议从 `.env.example` 复制：

```bash
cp .env.example .env
```

必须注意：

- `OPENAI_API_KEY` 使用占位值或部署环境变量，不写入仓库。
- `SESSION_STORAGE_DIR` 在生产环境应挂载到持久化目录。
- `OPENAI_BASE_URL` 可指向任意兼容 Chat Completions 的服务。

## Docker 部署要点

当前 docs 分支不修改 Docker 相关文件。如最终主分支提供 Dockerfile，可按以下原则部署：

1. 镜像内安装 Python 依赖与前端依赖。
2. 后端进程监听 `0.0.0.0:7860`。
3. 使用 `--env-file .env` 或平台密钥管理注入 LLM 配置。
4. 将宿主机 `data` 目录挂载到容器中的 session storage 路径。
5. 不把 `.env`、上传教材、session JSON 打包进镜像。

示例：

```bash
docker build -t kibot .
docker run --env-file .env -p 7860:7860 -v "$PWD/data:/app/data" kibot
```
