# KIBot 接口文档

基础地址：`http://localhost:7860`

## Health

### `GET /api/health`

返回服务状态。

```json
{
  "status": "ok",
  "service": "KIBot"
}
```

## Session

### `POST /api/session`

创建一个空 session。

### `GET /api/session/{session_id}`

读取 session 状态。`session_id` 必须是标准 UUID。

### `POST /api/session/{session_id}/reset`

重置指定 session，保留同一个 session id，清空教材、图谱、RAG、报告等状态。

## Textbooks

### `POST /api/textbooks/upload?session_id={session_id}`

上传并解析教材。也支持 form 字段 `session_id`。

请求：

- `multipart/form-data`
- `file`: `.pdf`、`.txt`、`.md`、`.markdown`、`.docx`
- 前端 `TextbookPanel` 支持拖拽上传、点击选择和批量选择；浏览器会对每个文件依次调用本接口，后端逐文件流式写入 `uploads` 并解析。

响应：`ParsedTextbook`，包含教材 id、文件名、标题、文件类型、页数、字符数、章节列表。

示例响应：

```json
{
  "textbook_id": "book_01",
  "filename": "02_组织学与胚胎学.pdf",
  "title": "组织学与胚胎学",
  "file_type": "pdf",
  "file_size_bytes": 73400320,
  "total_pages": 520,
  "total_chars": 385000,
  "chapters": [
    {
      "chapter_id": "ch_01",
      "title": "第一章 绪论",
      "page_start": 1,
      "page_end": 15,
      "content": "组织学与胚胎学研究机体微细结构与发生过程...",
      "char_count": 8500
    }
  ],
  "status": "parsed"
}
```

前端文件列表展示字段：文件名/标题、格式 `file_type`、大小 `file_size_bytes`、解析状态 `status`、章节数和字符数。上传过程中全局状态显示“上传解析中”；解析失败时接口返回 400，前端错误区展示失败原因。

### `GET /api/textbooks?session_id={session_id}`

列出当前 session 已上传教材。

### `POST /api/textbooks/{textbook_id}/select?session_id={session_id}`

将教材加入 `selected_textbooks`，供图谱和 RAG 使用。

### `DELETE /api/textbooks/{textbook_id}?session_id={session_id}`

删除教材，并清理对应章节、chunk 和选中状态。

## RAG

### `GET /api/rag/status?session_id={session_id}`

返回 RAG 准备状态、chunk 数、已选教材数、可检索 chunk 数和图谱节点数。

### `POST /api/rag/index`

兼容赛题参考接口，返回当前 session 的索引状态。KIBot 在教材上传时已经生成 chunk，并在查询时即时构建 BM25/hashed vector 检索结构，因此该接口用于显式验收“已索引 X 本教材，共 X 个知识块”。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000"
}
```

响应：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000",
  "indexed": true,
  "chunk_count": 1280,
  "selected_textbook_count": 7,
  "searchable_chunk_count": 1280,
  "message": "Chunks are indexed during textbook upload and selection."
}
```

### `POST /api/rag/query`

基于当前 session 的选中教材检索并回答。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000",
  "question": "请解释某个概念",
  "use_llm": false
}
```

也可使用 `query` 字段替代 `question`。`use_llm=false` 时返回 fallback 摘要；`use_llm=true` 且 LLM 配置可用时调用模型。

RAG pipeline 对应实现：

- 文档分块：`backend/services/chunker.py`，默认 `700` 字符窗口、`80` 字符 overlap，每个 chunk 保留教材名、章节、起止页码。
- Embedding / 检索：`backend/services/retriever.py`，BM25 + 本地 hashed vector embedding + 章节/概念 boost，默认 top-5。
- 生成约束：`_rag_messages()` 要求只使用 retrieved chunks，不足时说明信息不足，并用 `[1]` 形式引用证据。
- 前端证据：`RightTabs.tsx` 的 RAG 面板展示答案、引用列表、相关度分数，并可展开原文 source chunk。

响应字段包括：

- `answer`
- `answer_source`
- `retrieval_status`
- `citations`
- `source_chunks`
- `retrieved_chunks`
- `bm25_score`、`vector_score`、`chapter_score`、`concept_score`，位于每条 retrieved chunk 中
- `llm_error`，仅 LLM 调用失败时出现

完整响应示例：

## Integration

以下接口已实现，用于跨教材去重提纯、压缩统计和可视化整合流。

### `POST /api/integration/run`

读取选中教材、图谱和 chunk，生成跨教材融合决策并写入 `integration_decisions`。

自动整合执行 normalized name、containment、keyword overlap 语义对齐，输出 `merge` 与 `keep` 决策；教师可通过决策更新接口或对话接口把任一决策改为 `remove`、`merge`、`keep` 或 `split`。压缩统计由 `backend/services/integration_engine.py` 控制在 `ratio <= 0.30`。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000"
}
```

响应：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000",
  "decisions": [
    {
      "decision_id": "integration-merge-inflammatory-response",
      "action": "merge",
      "concept_name": "炎症反应",
      "sources": [
        {"textbook_title": "病理学", "chapter": "第四章 炎症"},
        {"textbook_title": "生理学", "chapter": "第九章 免疫"}
      ],
      "reason": "多本教材均描述同一防御反应，保留机制链路并去除重复定义。",
      "confidence": 0.88,
      "compact_note": "炎症反应：合并病理学的定义、生理学的介质调节和临床诊断学的体征描述。"
    }
  ],
  "stats": {
    "original_chars": 839000,
    "compressed_chars": 226530,
    "ratio": 0.27
  },
  "sankey": {
    "nodes": [{"name": "病理学"}, {"name": "整合-炎症反应"}],
    "links": [{"source": "病理学", "target": "整合-炎症反应", "value": 1}]
  }
}
```

### `GET /api/integration/decisions?session_id={session_id}`

返回当前 session 的融合决策列表。

### `POST /api/integration/decisions/{decision_id}`

教师更新某条决策的 action、note 或状态，并写回 session。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000",
  "action": "keep",
  "teacher_note": "教师要求保留该概念作为独立教学点。"
}
```

### `GET /api/integration/stats?session_id={session_id}`

返回压缩前后字符数、压缩率、合并/保留/冲突数量等统计。

### `GET /api/integration/sankey?session_id={session_id}`

返回整合流可视化需要的 Sankey 节点与边。

## Report

以下接口已实现，用于生成和读取 Markdown 整合报告。

### `POST /api/report/generate`

基于 session 中的教材、图谱、融合决策和引用证据生成 Markdown 报告。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000"
}
```

响应：

```json
{
  "markdown": "# KIBot 教材整合报告\n\n## 整合概览\n- 已选择教材：7 本\n- 压缩比例：27.00%\n...",
  "updated_at": "2026-05-10T07:02:45Z"
}
```

### `GET /api/report?session_id={session_id}`

读取 `report.markdown` 与 `report.updated_at`。

## Chat / 教师复核

### `POST /api/chat/message`

教师通过自然语言解释、保留、删除、合并或拆分整合决策。

请求：

```json
{
  "session_id": "00000000-0000-0000-0000-000000000000",
  "message": "keep 炎症反应，这个概念需要作为独立教学点保留"
}
```

响应：

```json
{
  "assistant_message": "Updated '炎症反应' to keep. I also updated the related graph status.",
  "parsed_intent": {"type": "keep", "concept": "炎症反应"},
  "state_summary": {
    "message_count": 2,
    "decision_count": 12,
    "report_updated_at": "2026-05-10T07:02:45Z"
  }
}
```

## 错误约定

- `400`：无效 session id、文件类型不支持、请求字段缺失。
- `404`：session 或教材不存在。
- LLM provider 错误会被转换为安全错误信息；检索接口仍尽量返回 fallback 答案。
