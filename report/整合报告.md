### 压缩率计算口径

本报告使用与后端 `backend/services/integration_engine.py` 一致的压缩率口径：

```text
original_chars = sum(selected_textbook.total_chars)
compressed_chars = sum(len(decision.compact_note) for decision in integration_decisions)
compression_ratio = compressed_chars / original_chars
```

代入本样例：

```text
original_chars = 839,000
compressed_chars = 226,530
compression_ratio = 226,530 / 839,000 = 0.2700 = 27.00%
```

其中 `original_chars` 表示 7 本教材解析后的原始正文字符量；`compressed_chars` 表示整合后保留下来的精华讲义字符量，即所有整合决策 `compact_note` 的总长度。该口径对应赛题中“把 7 本教材压缩到不超过原始体量 30%”的要求，并且能在 `/api/integration/stats` 和前端压缩指标中复核。
